Before you begin

How to compare the feedback

In each case, you will compare two anonymous feedback responses about the same machine translation. Your task is not to choose the better translation. Decide which feedback better helps a human reviewer understand the translation's quality and its important problems.

  1. Check the evidence Read the Source, Translation, and Reference. The Source is authoritative; the Reference is supporting evidence and may be imperfect.
  2. Evaluate correctness Prefer feedback that identifies real errors and explains them accurately. Penalize invented, exaggerated, unsupported, or contradictory claims.
  3. Find the key issue Prefer feedback that captures the most important problem affecting translation quality. Good feedback should not invent problems when the translation is acceptable.
  4. Judge overall usefulness Consider whether the feedback clearly explains why an issue matters and helps a reviewer understand the translation quality. Providing a rewritten translation is not required.

Avoid presentation bias

  • Do not prefer a response because it is longer or shorter.
  • Do not prefer a response because it uses tags, labels, headings, or elaborate formatting.
  • More detail is valuable only when it is correct and useful.
  • Judge each response by its content, not by how confident it sounds.

Choosing an answer

A/B is better
One response is meaningfully better for the current question.
Tie
Both responses are similarly useful or similarly acceptable.
Both poor
Both have substantial errors, unsupported claims, or omissions that make them unreliable or unhelpful. Do not use this merely because neither response is perfect.
